Apache Solr Ref Guide Proof of Concept

As part of the proposal to move the Ref Guide from Confluence, this site provides demos of a small section of the Solr Reference Guide converted to Asciidoc format and published with various options.

The source for these examples is available in the GitHub repo refguide-asciidoc-poc. See the README in particular for information about the methodologies used to produce these examples.

Since August 2016, several changes have been made to improve the conversion of pages from Confluence. The changes include: fixing code snippets and admonishments (such as NOTE, WARNING, etc.), auto-generated sidebar navigation, auto-generated PDF "base" file, and other smaller changes.

  • Demo replica of full Ref Guide: Uses Jekyll to generate HTML pages, including a comments section on each page, and sidebar navigation.
  • Sample PDF: Uses Asciidoctor tools to create a PDF. The PDF is created from the same set of pages used to generate the Jekyll HTML pages for direct comparison. Warning: it's currently 27Mb.

NOTE: Each example above is a demonstration of capabilities, not a final product ready for publication. Further effort is required before it is ready to replace Confluence.

Known Issues and ToDos

There are several remaining issues to be resolved that you may notice in these pages. See also the README in the in the GitHub repo refguide-asciidoc-poc for more details, and additional To Do items.

  • The PDF may still have some issues with heading levels and page breaks.
  • Nested blocks of content in tables (like code blocks or NOTEs, etc.) break the rows.
  • Nested ordered lists (like 1, 2, a, b, 3, 4) get converted as a bad quasi-ordered list (like 1, 2, 2, 2, 3, 4).
  • Finalize stylings of HTML and PDF outputs.
  • Sidebar nav is auto-generated, but only one level. A bit more work is needed to try deeper nesting.
  • The PDF may need to be compressed. Check this and investigate solutions once full Guide has been converted.
  • Fix broken "tag" pages in Jekyll site.
